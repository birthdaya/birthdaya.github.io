模板题

定义

  • 重儿子：父节点的所有儿子中子树节点表切最多的节点
  • 轻儿子：父节点中出重儿子以外的儿子
  • 重边：父节点和重儿子连成的边
  • 轻边：父节点和轻儿子连成的边
  • 重链：由多条重边连接而成的路径（叶子节点自己独立成为一条重链）

性质

  1. 整棵树会被剖分成若干条重链
  2. 轻儿子一定是每一条重链的顶点
  3. 任意一条路径被切分成不超过 lognlognlogn 条边

数据结构

  • fa[u]fa[u]fa[u] 存父节点
  • dep[u]dep[u]dep[u] 存 uuu 的深度
  • son[u]son[u]son[u] 存 uuu 的重儿子
  • sz[u]sz[u]sz[u] 存以 uuu 为根的子树的节点树 (注意：加上自己）
  • top[u]top[u]top[u] 存 uuu 所在重链的顶点

算法步骤

第一遍dfs

维护fa、dep、sonfa、dep、sonfa、dep、son数组

void dfs1(int u,int father){
	fa[u]=father,dep[u]=dep[father]+1,sz[u]=1;//设置父节点，深度，和子树节点数 
	for(auto v:e[u]){
	    if(v==father) continue; //如果到祖先就继续 
	    dfs1(v,u);//递归 
	    sz[u]+=sz[v]; //更新子树节点数 
	    if(sz[son[u]]<sz[v]) son[u]=v; //更新重儿子 
	}	
}

第二遍dfs

维护 toptoptop 数组，记录链头

void dfs2(int u,int x){
	top[u]=x;//记录链头 
	if(!son[u]) return; //不存在重儿子返回 
	dfs2(son[u],x);//搜重儿子 
	for(auto v:e[u]){
		if(v==fa[u] || v==son[u]) continue; //如果到自己的祖先或者重儿子就继续，不搜 
		dfs2(v,v);//搜轻儿子
	}
}

游标向上跳（与倍增相似）

让两个右边沿着各自的重链向上跳，跳到同一条重链上时，深度较小的游标所指向的点就是 LCA

int lca(int u,int v){
	while(top[u]!=top[v]){
		if(dep[top[u]]<dep[top[v]]) swap(u,v); //保证u在最底下 
		u=fa[top[u]]; 
	}
	if(dep[u]<dep[v]) return u; 
	return v;
}
